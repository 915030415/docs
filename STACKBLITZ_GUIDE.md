# 非开发人员：在线编写与实时预览手册 (StackBlitz)

如果您是非开发人员，不需要在电脑上安装任何开发环境（如 Node.js、Git 等），就可以直接在浏览器中编写文档并看到实时预览效果。

我们将使用 **StackBlitz** 工具来实现这一目标。

## 第一步：进入在线编辑器

1. 点击项目首页 (README.md) 中的 **"在浏览器中编辑文档"** 按钮（或直接访问项目提供的 StackBlitz 链接）。
2. StackBlitz 会自动克隆项目并配置环境。
3. **耐心等待**：页面右侧会出现一个终端窗口，它会自动运行 `pnpm install` 和 `npm start`。这可能需要 1-2 分钟。

## 第二步：实时预览效果

1. 当终端显示 `Docusaurus started successfully` 类似的字样后，右侧会自动打开一个浏览器预览窗口。
2. 如果预览窗口没有自动出现，请查看右侧是否有提示 "Open in New Tab" 或点击上方的 "Preview" 按钮。

## 第三步：如何修改文档

1. 在左侧的 **文件管理器** 中，找到 `docs` 目录。
2. 选择您想要修改的 `.md` 或 `.mdx` 文件。
3. 直接在中间的代码区域进行修改。
4. **即时反馈**：只要您修改并保存（通常是自动保存，或按下 `Ctrl + S` / `Cmd + S`），右侧的预览窗口会自动刷新，显示您最新的修改结果。

## 第四步：保存您的更改 (提交)

由于 StackBlitz 运行在您的浏览器中，您的更改需要“提交”回仓库才能永久保存：

1. 点击左侧工具栏的 **"Source Control" (源码管理图标)**。
2. 输入您的修改描述（例如：“更新了用户手册的第一章节”）。
3. 点击 **"Commit & Push"**（如果权限允许）或者 **"Create Pull Request"** 提交给开发人员。

---

### 常见问题 (FAQ)

- **问：预览窗口是空白的？**
  - 答：请检查右下角的终端是否有报错信息。如果没有报错，请尝试点击预览窗口顶部的刷新按钮。
- **问：修改了没有生效？**
  - 答：确认您修改的是 `docs` 目录下的文件，并且文件后缀是 `.md` 或 `.mdx`。
- **问：我不小心关掉了窗口怎么办？**
  - 答：如果您没有进行 Commit，您的更改可能不会保存在云端仓库，但 StackBlitz 通常会缓存您的本地修改。重新打开原来的链接即可。
